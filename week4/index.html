Single-Layer Perceptron (Rosenblatt 1957)

The perceptron is the simplest neural network: a binary linear classifier.

Discriminant function: f(x) = sign(w . x + b)

The decision boundary is a hyperplane in the input space. For 2D inputs, this is a straight line. The weight vector w is perpendicular to the decision boundary, and the bias b shifts it from the origin.

Output: +1 if w . x + b >= 0, else -1 (or 0/1 variant).

Linear Separability

A perceptron can only solve problems where the two classes can be separated by a single hyperplane (i.e., linearly separable).

No single line can separate the 1s from the 0s in XOR. This was famously shown by Minsky & Papert (1969) and contributed to the first "AI winter".

Perceptron Learning Rule

Weight update: ΔW = α(y - ŷ) · x

Where α is the learning rate, y is the target, and ŷ is the prediction.

Perceptron Convergence Theorem: If the training data is linearly separable, the perceptron learning algorithm is guaranteed to converge to a solution in a finite number of steps.

Key insight: If the prediction is correct, no weight update occurs. Updates only happen on misclassified examples.

Multi-Layer Perceptron (MLP)

An MLP consists of: Input layer → Hidden layer(s) → Output layer

Each layer is fully connected to the next. Hidden layers allow the network to learn non-linear decision boundaries.

Universal Approximation Theorem: An MLP with at least one hidden layer containing a sufficient number of neurons can approximate any continuous function to arbitrary precision.

This overcomes the linear separability limitation of single-layer perceptrons. XOR can be solved with one hidden layer.

Activation Functions

Activation functions introduce non-linearity. They must be differentiable for gradient-based learning.

Sigmoid: σ(s) = 1 / (1 + e^(-s)) — output in (0,1), derivative: σ(s)(1-σ(s))

Tanh: tanh(s) = (e^s - e^(-s)) / (e^s + e^(-s)) — output in (-1,1), zero-centered

ReLU: f(s) = max(0, s) — computationally efficient, avoids vanishing gradient for positive values

Sigmoid and tanh suffer from the vanishing gradient problem for very large or very small inputs. ReLU helps but can have "dead neurons" (always output 0).

Backpropagation

The algorithm for training MLPs by propagating errors backward through the network:

  1. Forward pass: Compute outputs layer by layer
  2. Compute error: Compare output to target (e.g., E = ½(target - output)²)
  3. Backward pass: Use the chain rule to compute ∂E/∂w for each weight
  4. Update weights: w ← w - α(∂E/∂w)

The chain rule is key: for a weight in an earlier layer, the gradient flows through all subsequent layers via multiplication of local gradients.

Gradient Descent

Update rule: w ← w - α(∂E/∂w)

The learning rate α controls step size:

  • Too large: oscillation or divergence
  • Too small: very slow convergence

Risk of local minima: The error surface may have multiple local minima. Gradient descent can get stuck in a local minimum rather than finding the global minimum.

Variants: Batch (full dataset), Stochastic (single sample), Mini-batch (subset).
