1. Introduction to Reinforcement Learning

What makes RL different from other types of learning?

What is Reinforcement Learning?

RL is a type of machine learning where an agent learns to make decisions by interacting with an environment. There is no teacher providing correct answers — the agent receives rewards (or penalties) as feedback and learns to maximise its cumulative reward over time.

The key distinction from other paradigms:

ParadigmInputSignal
Supervised LearningLabeled examples (x, y)Correct answer given for each input
Unsupervised LearningUnlabeled dataNo feedback at all — find structure
Reinforcement LearningStates & actionsDelayed scalar reward — learn by trial and error

Key idea: The RL agent has no labelled training data. It must discover which actions yield good outcomes through its own experience. This trial-and-error nature — combined with delayed rewards — is what makes RL fundamentally different.

2. Elements of Reinforcement Learning

The four building blocks: Policy, Reward, Value, Model

Four Key Components

Every RL system has (some or all of) these elements:

Policy (π)A mapping from states to actions. Tells the agent what to do in each situation. Can be deterministic or stochastic. Reward (R)A scalar signal received after each action. Defines the goal — the agent wants to maximise cumulative reward. Value Function (V)The expected total future reward from a given state. Unlike reward (immediate), value captures long-term desirability. ModelThe agent's internal representation of the environment (transition probabilities, expected rewards). Optional — model-free methods don't need one.

3. Exploration vs Exploitation

Greedy, ε-greedy, softmax — strategies for balancing the known and unknown

The Dilemma

Exploitation: Choose the action that currently seems best (highest estimated value).
Exploration: Try other actions to discover potentially better options.

Pure exploitation gets stuck on a locally good option. Pure exploration never commits. We need a balance.

Action-Value Estimation

For the multi-armed bandit problem, we estimate the value of each action:

Qt(a) = (sum of rewards when action a was taken) / (number of times a was taken)

Three strategies to select actions based on these estimates:

StrategyRuleTrade-off
GreedyAlways pick argmax Q(a)Pure exploitation — no exploration
ε-greedyWith prob ε, pick random; otherwise greedySimple, effective, most commonly used
SoftmaxP(a) ∝ eQ(a)/τTemperature τ controls exploration smoothly

4. The Agent-Environment Interface

States, actions, rewards, and the MDP formalism

The Interaction Loop

At each time step t, the agent:

  1. Observes state St
  2. Selects action At according to policy πt(a | s)
  3. Receives reward Rt+1
  4. Transitions to new state St+1
S0, A0, R1, S1, A1, R2, S2, A2, R3, ...

Markov Decision Process (MDP)

An MDP is defined by the 4-tuple <S, A, δ, r>:

SSet of states ASet of actions δTransition function: δ(s, a, s') = P(St+1 = s' | St = s, At = a) rReward function: r(s, a) = expected immediate reward after taking action a in state s
Markov Property: The future depends only on the current state, not on the history of how we got here. Formally: P(St+1 | St) = P(St+1 | S1, ..., St)

Markov Chain vs MDP

Markov ChainMarkov Decision Process
Actions?No — state transitions happen passivelyYes — agent chooses actions
Rewards?No reward signalReward after each transition
Defined by<S, δ><S, A, δ, r>
Use caseModelling sequences (weather, text)Decision-making under uncertainty
Key idea: The MDP framework <S, A, δ, r> formalises the RL problem. The Markov property means the current state contains all the information needed to predict the future.

5. Returns and the Discount Factor

Episodic vs continuing tasks, and why future rewards are worth less

Return Gt

The return is the total reward from time step t onward:

Gt = Rt+1 + γRt+2 + γ2Rt+3 + ... = ∑k=0∞ γk Rt+k+1

The discount factor γ ∈ [0, 1) controls how much the agent values future rewards:

  • γ → 0: Myopic — agent only cares about immediate reward
  • γ → 1: Far-sighted — agent weights future rewards almost equally
Episodic tasks have a natural end (games, mazes). Continuing tasks go on forever — discounting is essential to keep Gt finite.

Key idea: The discount factor γ serves two purposes: (1) it ensures the return is finite in continuing tasks, and (2) it encodes a preference for sooner rewards over later ones.

6. Value Functions

Vπ(s), Qπ(s,a), and the Bellman equation

State-Value Function Vπ(s)

The expected return when starting in state s and following policy π:

Vπ(s) = Eπ[Gt | St = s]

Action-Value Function Qπ(s, a)

The expected return when starting in state s, taking action a, and then following policy π:

Qπ(s, a) = Eπ[Gt | St = s, At = a]

Optimal Value Functions

The optimal state-value function V*(s) = maxπ Vπ(s) for all states.

The optimal action-value function gives the Bellman optimality equation:

Q*(s, a) = r(s, a) + γ ∑s' δ(s, a, s') · V*(s')

Or equivalently: the optimal policy greedily picks the action maximising Q*:

π*(s) = argmaxa Q*(s, a)

Key idea: V(s) rates how good a state is; Q(s,a) rates how good an action is from a state. The Bellman equation relates the value of a state to the values of its successor states.

7. Temporal-Difference Learning

Learning from incomplete episodes — the central idea of RL

Monte Carlo vs TD

Monte Carlo methods wait until the end of an episode to update V(St):

V(St) ← V(St) + α[Gt - V(St)]

TD(0) updates after every single step, using the next state's estimate as a bootstrap:

V(St) ← V(St) + α[Rt+1 + γV(St+1) - V(St)]

The term [Rt+1 + γV(St+1) - V(St)] is called the TD error (δ).

Try it: TD Update Step-by-Step

State A V=3.0
→ R = 1
State B V=5.0
1. TD target = R + γ × V(B) = 1 + 0.90 × 5.0 = 5.50
2. TD error = target - V(A) = 5.50 - 3.00 = 2.50
3. V(A) ← V(A) + α × TD error = 3.00 + 0.50 × 2.50 = 4.25
α = 0.50

Small α (e.g. 0.1)

Conservative updates. Old estimates are trusted more. Stable but slow to learn.

Large α (e.g. 0.9)

Aggressive updates. New information is trusted more. Fast but may be unstable.

Key idea: TD learning combines the bootstrapping idea (using estimates of successor states) with incremental updates. This lets the agent learn during an episode, not just at the end.

8. SARSA and Q-Learning

On-policy vs off-policy TD control

SARSA: On-Policy TD Control

Uses the 5-tuple (St, At, Rt+1, St+1, At+1) — hence the name.

Q(S, A) ← Q(S, A) + α[R + γQ(S', A') - Q(S, A)]

On-policy: it evaluates and improves the same policy it uses for action selection (e.g. ε-greedy).

Q-Learning: Off-Policy TD Control

Uses max over next actions instead of the action actually taken:

Q(S, A) ← Q(S, A) + α[R + γ maxa Q(S', a) - Q(S, A)]

Off-policy: it learns the optimal Q* regardless of the exploration policy being followed.

Key idea: SARSA and Q-learning differ in one critical way: SARSA uses Q(S', A') (the action actually taken), while Q-learning uses maxa Q(S', a) (the best possible action). This makes Q-learning converge to the optimal policy, while SARSA converges to the best policy given the exploration strategy.

9. From Q-Tables to Deep RL

Actor-Critic, PPO, and your Assignment 2

Why Neural Networks?

Q-tables work for small, discrete state spaces. But for large or continuous spaces (images, robot joints), we need function approximation. Deep Q-Networks (DQN) use a neural network to approximate Q(s, a) — achieving human-level performance on 49 Atari games (Mnih et al., 2015).

Actor-Critic Methods

Instead of a single Q-network, actor-critic architectures split the problem:

The TD error from the critic tells the actor how to improve: if an action led to a better-than-expected outcome, reinforce it.

PPO (Proximal Policy Optimization)

PPO is an actor-critic method that prevents the policy from changing too drastically in one update. It uses a clipped surrogate objective:

LCLIP(θ) = E[min(rt(θ) Ât, clip(rt(θ), 1-ε, 1+ε) Ât)]

where rt(θ) is the ratio of new to old policy probabilities, and Ât is the estimated advantage. The clipping keeps updates stable.

IPPO (Independent PPO)

Each agent has its own actor and critic. They learn independently, treating other agents as part of the environment.

MAPPO (Multi-Agent PPO)

Agents share a centralised critic that sees all agents' observations during training. Each agent still has its own actor. Better coordination through shared information.
