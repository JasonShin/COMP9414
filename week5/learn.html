Chapter 1: The Cookie-Finding Robot

How does a robot learn without a teacher?

🤖 ➡️ 🏠❓❓❓ ➡️ 🍪!

Imagine this...

You drop a little robot into a house it has never seen before. No map. No instructions. No idea where anything is.

Its only goal? Find the cookies! 🍪

The robot can only do one thing at a time: move up, down, left, or right. Every time it takes a step, its battery drains a tiny bit (-1). But when it finds a cookie... +10 points! 🎉

📸 Supervised Learning

Someone shows the robot 1000 photos of cookies and says "this is a cookie."

The robot learns to recognize cookies, but doesn't know how to find them.

🧑‍🏫 ➡️ 🤖 = "That's a cookie!"

🔍 Reinforcement Learning

Nobody tells the robot anything. It just wanders around, bumps into things, and eventually finds a cookie.

Then it thinks: "I should come back here!"

🤖 ➡️ 🚶 ➡️ 🚶 ➡️ 🍪 = "Yay!"

🔑 Key Vocabulary

Agent = the robot 🤖 — the thing that makes decisions

Environment = the house 🏠 — the world the agent lives in

State = which room the robot is in — its current situation

Action = move up/down/left/right — what the robot can do

Key Insight: In RL, there's no teacher showing the right answer. The agent learns by trying things and seeing what rewards it gets!

Chapter 3: The Discount Factor

Would you rather have 1 cookie NOW or 1 cookie TOMORROW?

🍪 today > 🍪 tomorrow > 🍪 next week?

🎁 Patience Has a Name: gamma (γ)

If someone offers you a cookie right now vs. a cookie next week, which do you pick? Most of us want it now!

In RL, we capture this with a number called gamma (γ), between 0 and 1:

  • γ = 0 means "I only care about RIGHT NOW" (super impatient!)
  • γ = 0.99 means "I almost equally value future rewards" (super patient!)
  • γ = 0.5 means "each future step is worth HALF as much"

🎮 Adjust γ and watch the cookies change!

The math: Today = 1.0, Tomorrow = γ, Day after = γ2, then γ3, γ4...

γ = 0.50
Key Insight: γ controls how far into the future the agent looks. Low γ = greedy/short-sighted. High γ = patient/far-sighted.

Chapter 4: Explore or Exploit?

The great dilemma: stick with what works, or try something new?

🗺️ Try new things?   or   💰 Stick with what works?

🍕 The Restaurant Dilemma

You found a pizza place that's 7/10. Pretty good!

But there's a new restaurant next door. It might be 9/10... or it might be 2/10. Do you risk it?

💰 Exploit

Always go to your best-known option. Safe, but you might miss something amazing!

🗺️ Explore

Sometimes try random things. Risky, but you might discover something great!

Chapter 5: Learning Step by Step

TD Learning: updating your guesses on the go

🤖👣 "I don't need to arrive to know I'm on the right track!"

🏫 The Walk to School Analogy

Imagine walking to school. You don't need to arrive to know you're going the right way — seeing familiar landmarks tells you!

TD Learning works the same way. The robot updates its guess every single step, not just when it finds the cookie.

🧮 The TD Update (step by step)

Cell A V=3.0
➡️ r = 1
Cell B V=5.0
1. New estimate of A = reward + γ x V(B)
    = 1 + 0.90 x 5.0 = 5.50

2. TD Error = new estimate - old estimate
    = 5.50 - 3.00 = 2.50

3. V(A) ← old + α x TD Error
    = 3.00 + 0.50 x 2.50 = 4.25

🎮 Adjust α (learning rate) and see what happens!

α controls how much the robot trusts the new information vs. its old belief.

α = 0.50

α = small (0.1)

🐢 Slow learner. Trusts old beliefs. Barely moves toward new info. Stable but slow.

α = large (0.9)

🐇 Fast learner. Trusts new info immediately. Quick but might overshoot!

Key Insight: TD Learning lets the agent learn during the journey, not just at the end. It updates its value estimates step by step using the formula: V(s) ← V(s) + α [r + γV(s') - V(s)]

Chapter 6: Q-Table: Rating Every Move

Not just "how good is this room?" but "how good is going LEFT from this room?"

📊 Q(state, action) = "How good is doing THIS from HERE?"

From Values to Q-Values

Before, we rated each room. Now we rate each (room, direction) pair!

Q(kitchen, left) = 8.5 means "going left from the kitchen is awesome!" 🎉

Q(kitchen, right) = -2.0 means "going right from the kitchen is bad!" 😬

🎮 Train the Q-Table!

A 3x4 gridworld: 🏆 = goal (+1), 💀 = pit (-1). Click "Train 1 Episode" to watch the robot learn!

Episodes trained: 0

Key Insight: A Q-table stores a value for every (state, action) pair. The agent picks the action with the highest Q-value. As it trains more, the values converge to the optimal policy!

Chapter 7: SARSA vs Q-Learning

The Careful Turtle vs. the Bold Rabbit

🐢 "I'll take the safe path"    🐇 "I'll take the shortcut!"

🏔️ The Cliff World

Both robots start at S and want to reach G. The bottom row is a cliff (instant -100 and restart!).

The safe path goes around the top. The risky (but shorter) path goes along the cliff edge.

Episodes: 0

🐢 SARSA (On-Policy)

"What will I actually do next (including mistakes)?"

Learns the safe path because it accounts for its own random exploration near the cliff.

Avg reward: 0

🐇 Q-Learning (Off-Policy)

"What's the best I could do next?"

Learns the optimal path close to cliff (risky during training but optimal in theory).

Avg reward: 0

Key Insight: SARSA uses the action it actually takes next (on-policy). Q-Learning uses the best possible next action (off-policy). This makes SARSA safer but Q-Learning closer to optimal.

📦 PPO, IPPO, and MAPPO

PPO (Proximal Policy Optimization)
A smarter version of policy gradient. It says: "don't change the policy too much in one step" by clipping the update. Think of it as training wheels for the learning process!

IPPO (Independent PPO)

Each robot learns independently — like 3 separate Q-learners that don't talk to each other.

🤖   🤖   🤖
Each has own Actor + Critic

MAPPO (Multi-Agent PPO)

Robots share a critic — during training, one brain can see what ALL robots see. Better coordination!

🤖🤖🤖
Shared Critic, separate Actors
